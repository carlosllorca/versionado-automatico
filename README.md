# Versionado autmático

En proyectos JavaScript, estandarizar el control de versiones y automatizar procesos clave es fundamental para mantener
la calidad y coherencia del desarrollo. Herramientas como Husky, que ejecuta validaciones en los hooks de Git; 
semantic-release, que gestiona automáticamente el versionado y la generación de changelogs; y GitHub Actions,
que permite integrar estos procesos en un flujo CI/CD, ofrecen una solución robusta y escalable. Esta guía explica
cómo combinar estas herramientas para crear un sistema automatizado que valida commits, publica versiones siguiendo 
el versionado semántico y mantiene el proyecto siempre listo para producción, reduciendo errores humanos y mejorando la
eficiencia del equipo.
Con este repositorio pretendo ejmplificar la integración de estas herramientas en un proyecto de ejemplo con NextJS.
